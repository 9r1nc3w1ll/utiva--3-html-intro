Why Use Semantic HTML?

Semantic HTML elements clearly describe their meaning to both the browser and the developer. Benefits include:

  • Better accessibility for screen readers and assistive technologies
  • Improved SEO as search engines understand your content better
  • More maintainable code with clearer structure
  • Easier styling with CSS

<header> Element

The header element represents introductory content for its nearest ancestor sectioning content or sectioning root element.

This is a header within an article. Headers can be used in articles, sections, or for the page itself.

A header typically contains:

  • One or more heading elements (<h1>-<h6>)
  • Logo or icon
  • Authorship information

<nav> Element

The nav element represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents.

Common examples of navigation sections are:

  • Menus
  • Tables of contents
  • Indexes

<main> Element

The main element represents the dominant content of the body of a document. The main content area consists of content that is directly related to or expands upon the central topic of a document.

Important notes about <main>:

  • There should be only one <main> element in a document
  • It should not be a descendant of <article>, <aside>, <footer>, <header>, or <nav>

<section> Element

The section element represents a generic section of a document or application. A section, in this context, is a thematic grouping of content, typically with a heading.

Use <section> when:

  • The content can be naturally grouped under a heading
  • You'd list the section in a document outline or table of contents
  • The content represents a thematic grouping

<article> Element

The article element represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable.

Articles are commonly used for:

  • Forum posts
  • Blog posts
  • News stories
  • User comments
  • Product cards

An article should make sense on its own, even if removed from the rest of the site.

<aside> Element

The aside element represents a section of a page that consists of content that is tangentially related to the content around it, which could be considered separate from that content.

Common uses of <aside>:

  • Sidebars
  • Pull quotes
  • Advertising
  • Related article links
  • Author bios

<figure> and <figcaption> Elements

The figure element represents self-contained content, potentially with an optional caption, that is referenced from the main content.

Figures are often used for:

  • Images with captions
  • Charts and graphs
  • Code snippets
  • Quotations
  • Diagrams

<footer> Element

The footer element represents a footer for its nearest ancestor sectioning content or sectioning root element.

Footer typically contains:

  • Authorship information
  • Copyright information
  • Contact information
  • Sitemap
  • Back to top links
  • Related documents
