Responsive Images in HTML

Basic Responsive Image

Using CSS to make images responsive (max-width: 100% and height: auto):

<img src="image.jpg" alt="Description" /> /* CSS */ img { max-width: 100%; height: auto; }

This approach ensures the image is never wider than its container or original size.

Responsive Images with srcset (Different Sizes)

Using the srcset attribute with width descriptors (w):

<img srcset="small.jpg 400w, medium.jpg 800w, large.jpg 1200w" sizes="(max-width: 600px) 400px, (max-width: 1000px) 800px, 1200px" src="medium.jpg" alt="Responsive image example" />

The browser will choose the appropriate image based on:

  • The viewport width
  • The device's pixel density
  • The sizes attribute conditions

Responsive Images with srcset (Different Densities)

Using the srcset attribute with pixel density descriptors (x):

<img srcset="image.jpg 1x, image-2x.jpg 2x" src="image.jpg" alt="Image for different screen densities" />

This is especially useful for high-resolution displays (like Retina displays). The browser will select the image based on the device's pixel ratio.

Art Direction with the picture Element

Using the picture element to provide different images based on viewport:

<picture> <source media="(min-width: 800px)" srcset="desktop.jpg" /> <source media="(min-width: 600px)" srcset="tablet.jpg" /> <img src="mobile.jpg" alt="Art direction example" /> </picture>

The picture element allows you to specify different images for different viewport sizes. This is not just about resolution, but about actually showing different compositions or aspect ratios based on screen size.

Image Format Fallbacks

Using the picture element to provide modern formats with fallbacks:

<picture> <source type="image/webp" srcset="image.webp" /> <source type="image/jpeg" srcset="image.jpg" /> <img src="image.jpg" alt="Format fallback example" /> </picture>

This approach allows you to serve modern, efficient formats like WebP to browsers that support them, while providing JPEG fallbacks for older browsers.
